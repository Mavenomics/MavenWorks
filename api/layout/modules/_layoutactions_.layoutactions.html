SurroundWith

  • SurroundWith(__namedParameters: object): string
    • Defined in layoutactions.ts:42

    Surround a layout region in-place with a new parent.

    throws

    If the target ID does not exist in the layout.

    throws

    If the target region doesn't have a parent region.

    Notes

    This won't check to see if the parent is already of the given type. That is, it will always surround with the given region. Callers that use this function automatically should first check to see that the parent is of the type they need.

    Parameters

    • __namedParameters: object
      • layoutManager: LayoutManager

        The manager controlling the layout.

      • prunable: boolean
      • regionType: ContainerTypes

        The type of region to add.

      • target: string

        The ID of the region to surround.

    Returns string

    Conditionally surround a region if it is not already a container.

    If the target is an instance of the RegionClass, then it is considered a match and the target is returned without modification.

    If propsMustMatch is provided, each [property, value] pair will be compared with the target. If any target[[property]] !== value, then target will not be considered a match.
